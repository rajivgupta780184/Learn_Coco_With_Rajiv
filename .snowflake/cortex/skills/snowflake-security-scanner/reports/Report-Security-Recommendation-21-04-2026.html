Detailed Recommendations

P0 R1: Enable data exfiltration prevention

EXFIL_PREVENTION_DISABLEDEffort: Low

ALTER ACCOUNT SET PREVENT_UNLOAD_TO_INLINE_URL = TRUE;
ALTER ACCOUNT SET PREVENT_LOAD_FROM_INLINE_URL = TRUE;
ALTER ACCOUNT SET REQUIRE_STORAGE_INTEGRATION_FOR_STAGE_CREATION = TRUE;
ALTER ACCOUNT SET REQUIRE_STORAGE_INTEGRATION_FOR_STAGE_OPERATION = TRUE;
Risk: Existing COPY INTO workflows using inline URLs will break. Audit QUERY_HISTORY first.

P0 R2: Enable Trust Center & all scanner packages

TRUST_CENTER_NOT_ENABLEDEffort: Low

Trust Center provides automated CIS Benchmark scanning, Security Essentials hygiene checks, and Threat Intelligence detection. Without it, no automated security scanning is running.

Step 1 — Enable Trust Center (via Snowsight UI):

Navigate to Monitoring → Trust Center and click Enable. This requires ACCOUNTADMIN role.

Step 2 — Enable all scanner packages:

CALL SNOWFLAKE.TRUST_CENTER.ENABLE_SCANNER_PACKAGE('SECURITY_ESSENTIALS');
CALL SNOWFLAKE.TRUST_CENTER.ENABLE_SCANNER_PACKAGE('CIS_BENCHMARKS');
CALL SNOWFLAKE.TRUST_CENTER.ENABLE_SCANNER_PACKAGE('THREAT_INTELLIGENCE');

Step 3 — Verify scanners are running:

SELECT scanner_package_name, scanner_name, is_enabled
FROM SNOWFLAKE.TRUST_CENTER.SCANNER_PACKAGES_VIEW
ORDER BY scanner_package_name;
Impact: Once enabled, Trust Center will automatically surface findings for CIS controls, detect threats, and provide remediation guidance — covering many of the gaps identified in this assessment.

P1 R3: Create enterprise password policy

NO_PASSWORD_POLICYEffort: Low

CREATE OR REPLACE PASSWORD POLICY enterprise_password_policy
    PASSWORD_MIN_LENGTH = 14 PASSWORD_MAX_LENGTH = 256
    PASSWORD_MIN_UPPER_CASE_CHARS = 2 PASSWORD_MIN_LOWER_CASE_CHARS = 2
    PASSWORD_MIN_NUMERIC_CHARS = 2 PASSWORD_MIN_SPECIAL_CHARS = 2
    PASSWORD_MIN_AGE_DAYS = 1 PASSWORD_MAX_AGE_DAYS = 90
    PASSWORD_MAX_RETRIES = 5 PASSWORD_LOCKOUT_TIME_MINS = 30
    PASSWORD_HISTORY = 12
    COMMENT = 'Enterprise password policy';
ALTER ACCOUNT SET PASSWORD POLICY = enterprise_password_policy;

P1 R4: Create session policies

NO_SESSION_POLICYEffort: Low

CREATE OR REPLACE SESSION POLICY standard_session_policy
    SESSION_IDLE_TIMEOUT_MINS = 60 SESSION_UI_IDLE_TIMEOUT_MINS = 30
    COMMENT = 'Standard - 60 min idle, 30 min UI idle';
ALTER ACCOUNT SET SESSION POLICY = standard_session_policy;

P1 R5: Create restrictive network policy

NO_NETWORK_POLICYEffort: Medium

Step 1 — Identify legitimate IPs from login history. Step 2 — Create allowlist:

CREATE OR REPLACE NETWORK POLICY corporate_network_policy
    ALLOWED_IP_LIST = ('<your_office_ip>/32', '<your_vpn_ip>/32')
    COMMENT = 'Restrictive corporate network policy';

Step 3 — Test on one user, then apply account-wide:

ALTER USER RAJIVGUPTA780184 SET NETWORK_POLICY = corporate_network_policy;
-- After validation:
ALTER ACCOUNT SET NETWORK_POLICY = corporate_network_policy;
Risk: If current IP not in allowlist, you'll be locked out. Always include your current IP.

P1 R6: Enforce MFA for all password logins

PASSWORD_LOGIN_NO_MFAEffort: Low

CREATE OR REPLACE AUTHENTICATION POLICY enforce_mfa_policy
    MFA_ENROLLMENT = 'REQUIRED'
    MFA_AUTHENTICATION_METHODS = ('TOTP')
    CLIENT_TYPES = ('SNOWFLAKE_UI', 'SNOWSIGHT', 'DRIVERS', 'SNOWSQL')
    AUTHENTICATION_METHODS = ('PASSWORD', 'SAML')
    COMMENT = 'Requires MFA for all password logins';
ALTER ACCOUNT SET AUTHENTICATION POLICY = enforce_mfa_policy;

P2 R7: Separate admin roles

MULTI_ADMIN_ROLESEffort: Medium

CREATE ROLE IF NOT EXISTS PLATFORM_ADMIN_ROLE;
GRANT ROLE SYSADMIN TO ROLE PLATFORM_ADMIN_ROLE;
GRANT ROLE PLATFORM_ADMIN_ROLE TO USER RAJIVGUPTA780184;
ALTER USER RAJIVGUPTA780184 SET DEFAULT_ROLE = 'PLATFORM_ADMIN_ROLE';

P2 R8: Enable periodic data rekeying (CIS 4.9)

PERIODIC_REKEYING_OFFEffort: Low

ALTER ACCOUNT SET PERIODIC_DATA_REKEYING = TRUE;
This ensures data encrypted more than one year ago is re-encrypted with newer keys. No impact on query performance.

P2 R9: Implement data masking policies (CIS 4.10)

NO_MASKING_POLICIESEffort: Medium

-- Example: email masking
CREATE OR REPLACE MASKING POLICY pii_email_mask AS (val STRING) RETURNS STRING ->
    CASE WHEN CURRENT_ROLE() IN ('DATA_ADMIN', 'ACCOUNTADMIN') THEN val
         ELSE REGEXP_REPLACE(val, '.+@', '****@') END;

-- Apply to sensitive columns:
ALTER TABLE <DB>.<SCHEMA>.<TABLE> MODIFY COLUMN EMAIL SET MASKING POLICY pii_email_mask;

P2 R10: Implement row-access policies (CIS 4.11)

NO_ROW_ACCESS_POLICIESEffort: Medium

-- Example: department-based row filtering
CREATE OR REPLACE ROW ACCESS POLICY dept_row_filter AS (dept_col VARCHAR) RETURNS BOOLEAN ->
    CURRENT_ROLE() IN ('ACCOUNTADMIN') OR dept_col = CURRENT_ROLE();

ALTER TABLE <DB>.<SCHEMA>.<TABLE> ADD ROW ACCESS POLICY dept_row_filter ON (DEPARTMENT);
